Lesson 3 Google Maps API

Technology Stack

This project was built using Visual Studio 2015 and the ASP.NET framework. This project is being served via IIS running on a AWS t.2 micro instance. The Google Maps API is being used to deliver the mapping content. Bootstrap was used as the framework for the webpage with credit given to bootstrapzero for a starting template.

The Process

The largest challange I encountered was figuring out how to get the Google Maps API to work with Bootstrap. Initially, I tried to use the JavaScript code direct from the Google API page, but the map would not render inside the Bootstrapped page. If I removed the Bootstrap functionality from the page everything worked fine. After some research, I discovered that I needed to create an event listener that would initialize the function after the page had finished loading. After setting up the listener, I no longer had any issues. A smalller issue that I encountered was making the pages respond appropriately for its intended destination. I originally developed the page on a very large monitor, but it was intended to be displayed on a 980px wide section in my portfolio. This was the main reason for using the bootstrap framework. I was able to develop a suitable compromise for both my large screen monitor and a smaller display.

I also experiement with a JQuery API titled gmap3. I eventually commneted this code out from the project, but I found the gmap3 API much easier to work with and will likely build all of my professional maps with this API in the future. I downloaded the JQuery plugin from GitHub . If you would like to see the code for this project you could examine the developer tools or it can be found here on GitHub.
